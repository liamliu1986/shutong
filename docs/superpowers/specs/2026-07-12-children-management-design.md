# 前端孩子管理功能设计文档

**日期：** 2026-07-12
**主题：** 实现前端孩子管理页面

---

## 1. 背景与目标

后端已提供完整的孩子管理 API：
- `GET /users/me/children`：获取当前用户的孩子列表
- `POST /users/me/children`：添加孩子
- `DELETE /users/me/children/{child_id}`：删除孩子

但前端目前没有对应的管理页面，导致用户无法在前端创建孩子，进而影响需要选择孩子的功能（如添加错题、错题列表筛选等）。

本次目标：实现前端孩子管理页面，使用户可以查看、添加、删除孩子。

---

## 2. 设计概述

- **页面路径：** `/children`
- **页面类型：** Next.js 14 App Router Client Component
- **布局：** 使用现有 `MainLayout`
- **API：** 复用 `frontend/src/lib/api.ts` 中的 `childrenAPI`

页面包含两个核心区域：
1. 添加孩子表单（顶部）
2. 孩子列表（下方）

---

## 3. 页面结构

### 3.1 文件：`frontend/src/app/children/page.tsx`

#### 添加孩子表单

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 姓名 | text | 是 | 1-50 字符 |
| 年级 | select | 是 | 7-12，对应初一至高三 |
| 科目 | checkbox 组 | 否 | 数学、物理、化学、英语 |

年级映射：
- 7：初一
- 8：初二
- 9：初三
- 10：高一
- 11：高二
- 12：高三

#### 孩子列表

以卡片网格展示，每张卡片显示：
- 姓名
- 年级（中文标签，如「高一」）
- 科目标签
- 创建时间
- 删除按钮

#### 状态处理

- 页面加载中：显示加载提示
- 加载失败：显示错误信息 + 重试按钮
- 无孩子：显示空状态提示
- 添加中：禁用提交按钮
- 删除前：弹出 `window.confirm` 确认框

---

## 4. 数据流

```
用户访问 /children
  → useAuth 验证登录状态
  → 调用 childrenAPI.getChildren()
  → 渲染孩子列表

用户填写表单并提交
  → 调用 childrenAPI.createChild({ name, grade, subjects })
  → 成功后：刷新列表、清空表单
  → 失败后：显示错误信息

用户点击删除
  → window.confirm("确定删除该孩子吗？")
  → 确认后调用 childrenAPI.deleteChild(childId)
  → 成功后：刷新列表
  → 失败后：显示错误信息
```

---

## 6. 错误处理

| 场景 | 处理方式 |
|---|---|
| 未登录 | 显示「请先登录」提示 |
| 加载失败 | 显示错误信息 + 重试按钮 |
| 姓名为空 | HTML5 `required` 拦截 |
| 后端返回 422 | 显示校验错误详情 |
| 删除确认取消 | 不执行任何操作 |
| 删除失败 | 显示错误提示 |
